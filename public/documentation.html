API Documentation

Endpoints

The following table provides details about the API endpoints, including their request methods, URLs, and expected responses.

Movies

Request URL Response
GET /movies Retrieve a list of all movies.
Example: [{"id": 1, "title": "Jab We Met", "genre": "Romance"}]
GET /movies/{id} Retrieve details of a single movie by ID.
Example: {"id": 1, "title": "Jab We Met", "genre": "Romance"}
POST /movies Create a new movie.
Request Body: {"title": "Tamasha", "genre": "Drama"}
Response: {"id": 2, "message": "Movie created successfully."}
PUT /movies/{id} Update an existing movie by ID.
Request Body: {"title": "Jab We Met", "genre": "Drama"}
Response: {"message": "Movie updated successfully."}
DELETE /movies/{id} Delete a movie by ID.
Response: {"message": "Movie deleted successfully."}

Users

Request URL Response
GET /users Retrieve a list of all users.
Example: [{"id": 1, "name": "Alice", "email": "alice@example.com"}]
GET /users/{id} Retrieve details of a single user by ID.
Example: {"id": 1, "name": "Alice", "email": "alice@example.com"}
POST /users Create a new user.
Request Body: {"name": "Bob", "email": "bob@example.com"}
Response: {"id": 2, "message": "User created successfully."}
PUT /users/{id} Update an existing user by ID.
Request Body: {"name": "Alice Johnson", "email": "alice.j@example.com"}
Response: {"message": "User updated successfully."}
DELETE /users/{id} Delete a user by ID.
Response: {"message": "User deleted successfully."}
